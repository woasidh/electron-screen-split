# Drag Stability Design

## 문제

- 상태 이벤트가 올 때마다 슬롯 카드 DOM 전체를 교체함.
- 드래그 원본 인덱스가 렌더 함수 지역 변수라 재렌더링 후 사라짐.
- 드래그 도중 원본 노드가 제거되어 드롭이 간헐적으로 무시됨.

## 결정

- 드래그 원본 인덱스를 슬롯 그리드 컨테이너에 저장해 카드 재생성 후에도 유지함.
- 드래그 시작부터 종료까지 슬롯 그리드 렌더링을 차단함.
- 차단 중 들어온 마지막 렌더 요청을 기억했다가 드래그 종료 직후 한 번 실행함.
- 편집기, 출력 정보, 요약 상태는 기존처럼 갱신함.

## 범위

- Tauri 관리 화면의 슬롯 카드 드래그 교환만 수정함.
- 슬롯 저장, 출력 웹뷰 배치, 기존 Electron 앱은 변경하지 않음.
- Ubuntu 24.04와 Windows 10에서 공통으로 쓰는 표준 HTML 드래그 이벤트를 유지함.

## 검증

- 드래그 도중 카드 렌더 함수를 다시 호출해도 원본/대상 교환 콜백이 실행되는지 테스트함.
- 렌더 차단 중 요청이 실행되지 않고 차단 해제 시 한 번만 실행되는지 테스트함.
- 전체 프런트엔드 테스트, 타입 검사, 번들 빌드, Rust 테스트를 실행함.
- 실행 중인 Tauri 관리 화면에서 실제 드래그 후 위치가 교환되는지 확인함.
