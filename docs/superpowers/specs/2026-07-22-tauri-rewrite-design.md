# Tauri Screen Wall Rewrite Design

## 목표

기존 Electron 기반 Screen Wall Control을 Tauri 2 기반으로 재작성한다. 관리 화면에서 네 개의 원격 URL과 확대율·사용 여부·배치를 관리하고, 주 모니터 전체를 네 개의 독립 웹뷰로 정확히 2×2 분할 출력한다.

## 지원 환경

- Tauri Rust crate는 `2.11.x` minor 버전으로 고정하고 `Cargo.lock`을 커밋
- macOS 14 이상, arm64 및 x64
- Windows 10 이상, x64
- Ubuntu 24.04, x64
- Ubuntu의 Wayland와 X11 세션 모두 검증
- 목표 출력 해상도 3840×2160
- 화면 배율 100%, 125%, 150%, 200% 검증

## 확정된 제품 정책

- 출력 모니터는 주 모니터로 고정한다.
- 웹 미리보기와 5초 자동 캡처 기능은 제거한다.
- 네 슬롯의 쿠키·캐시·로컬 스토리지는 슬롯별로 분리한다.
- 실행 중 ESC 안내 오버레이를 유지한다.
- URL 로딩에 실패한 슬롯은 검정 화면으로 남기고 정상 슬롯은 계속 출력한다.
- 원격 웹뷰의 오디오는 음소거한다.
- 기존 Electron `config.json`은 최초 실행 시 자동 이전한다.
- 원격 웹뷰에는 Tauri 시스템 권한을 부여하지 않는다.

## 접근법 비교

### 선택: 하나의 출력 Window와 네 개의 child Webview

- 장점: 한 창 안에서 경계 없는 2×2 출력, 전체화면·포커스·복귀 상태를 한곳에서 관리 가능
- 단점: Tauri의 `unstable` child webview API 사용 필요, Linux에서 GTK 전용 배치 보정 필요 가능

### 대안: 네 개의 WebviewWindow

- 장점: 공개 API 위주로 구현 가능, 각 웹뷰 수명주기 단순
- 단점: 창 사이 경계·포커스·항상 위·전체화면·작업표시줄 처리가 복잡하므로 제외

### 제외: 하나의 웹뷰 안에 네 개의 iframe

- 외부 사이트의 `X-Frame-Options` 및 CSP에 의해 차단될 수 있음
- 슬롯별 브라우징 데이터 분리와 장애 격리가 어려우므로 제외

## 애플리케이션 구조

### 관리 화면

- Vanilla TypeScript와 Vite로 구성한다.
- 현재 관리 화면의 URL 입력, 확대율, 사용 여부, 드래그 위치 교환, 상태 표시, RUN 동작을 유지한다.
- 미리보기 이미지는 제거하고 각 슬롯을 URL·위치·활성 상태·로딩 상태 카드로 표시한다.
- 프런트엔드는 Tauri command로 초기 상태 조회, 설정 저장, 실행, 중지를 요청한다.
- Rust가 보내는 출력 정보와 슬롯 상태 이벤트만 구독한다.

### Rust 코어

- `config`: 설정 정규화, 유효성 검사, 원자적 저장, Electron 설정 이전 담당
- `layout`: 물리 픽셀 기준 2×2 좌표와 확대율 계산 담당
- `wall`: 출력 Window와 네 child Webview의 생성·배치·실행·정지·복구 담당
- `webview`: 플랫폼별 세션 분리, 음소거, Linux GTK 배치 보정 담당
- `commands`: 관리 웹뷰에서 호출 가능한 최소 명령만 노출
- `events`: 슬롯 상태와 출력 환경 변경을 관리 화면으로 전달

## 출력 Window와 웹뷰

- 장식·리사이즈·작업표시줄 항목이 없는 검정색 Window를 생성한다.
- 주 모니터의 물리 해상도와 위치로 Window 크기를 설정한다.
- macOS는 simple fullscreen, Windows와 Ubuntu는 fullscreen을 사용한다.
- Window는 실행 중 항상 위 상태를 유지한다.
- 네 child Webview는 물리 픽셀 기준 좌상·우상·좌하·우하 영역을 빈 픽셀 없이 채운다.
- Tauri 모니터 크기는 이미 물리 픽셀이므로 해상도 판정 시 `scaleFactor`를 다시 곱하지 않는다.
- child Webview 생성·크기 API에 논리 좌표가 필요하면 명시적으로 `physical / scaleFactor` 변환을 수행한다.
- 설정 확대율은 각 플랫폼에서 페이지 CSS viewport가 기존 Electron 출력과 같아지도록 적용한다.
- 비활성 또는 실패 슬롯은 `about:blank` 검정 화면을 표시한다.

## Ubuntu 배치 전략

- WebKitGTK 4.1과 GTK 3 기반으로 구현한다.
- Wayland와 X11에서 동일한 GTK 컨테이너 경로를 사용한다.
- Tauri 기본 `GtkBox`에 생성된 child webview를 전용 GTK `Fixed` 컨테이너로 재부모화한다.
- Linux 배치는 GTK `Fixed::move_`와 크기 요청으로 적용하고, 다른 OS는 Tauri/Wry `set_bounds`를 사용한다.
- 화면 크기 변경 시 네 웹뷰를 재배치하고, 재배치 실패 시 출력 Window와 웹뷰를 재생성한다.
- NVIDIA 환경의 빈 화면을 별도 테스트하며 필요한 경우 `WEBKIT_DISABLE_DMABUF_RENDERER=1` 실행 옵션을 패키지 런처에 적용한다.

## 세션과 브라우징 데이터

- 슬롯 ID `wall-slot-1`부터 `wall-slot-4`까지를 브라우징 데이터 키로 사용한다.
- Windows와 Ubuntu는 슬롯별 데이터 디렉터리를 사용한다.
- macOS는 슬롯별 `dataStoreIdentifier`를 사용한다.
- 설정 카드 위치를 교환해도 세션은 화면 위치에 남는다. 즉 URL 설정은 이동하지만 해당 URL의 로그인 상태는 새 위치 세션을 사용한다.
- 출력 웹뷰를 제거했다가 다시 만들더라도 동일 슬롯 키를 사용해 로그인 상태를 유지한다.

## ESC, 오버레이, 커서

- 실행 직전에 Rust에서 Tauri global-shortcut 플러그인으로 Escape를 등록한다.
- 복귀 시 Escape 등록을 즉시 해제한다.
- 등록에 실패하면 오버레이를 클릭해 관리 화면으로 복귀할 수 있게 한다.
- 오버레이는 마지막 child Webview로 생성해 네 원격 웹뷰 위에 표시한다.
- 포인터 위치를 Rust에서 주기적으로 비교해 이동을 감지한다.
- 포인터 이동 시 오버레이와 커서를 표시하고 3초 입력이 없으면 둘 다 숨긴다.
- macOS simple fullscreen 해제, 출력 Window 닫기, 두 번째 앱 실행도 관리 화면 복귀로 처리한다.

## 오디오 정책

- Windows는 WebView2 네이티브 음소거 기능을 우선 사용한다.
- Ubuntu와 macOS는 문서 시작 스크립트로 기존 및 새 `audio`·`video` 요소를 음소거한다.
- 같은 문서 시작 스크립트에서 `AudioContext` 생성을 감싸 새 컨텍스트를 즉시 suspend한다.
- 페이지 탐색이 완료될 때마다 음소거 상태를 다시 적용한다.

## 보안

- 관리 웹뷰와 오버레이 웹뷰만 각각 필요한 최소 capability를 가진다.
- 네 원격 웹뷰는 Tauri capability 대상에서 제외한다.
- 모든 custom command는 호출 웹뷰 label이 관리 웹뷰인지 재검증한다.
- 원격 URL은 HTTP와 HTTPS만 허용한다.
- 새 창 요청과 다운로드 요청은 거부한다.
- 설정 문자열은 길이를 제한하고 저장 전에 Rust에서 다시 정규화한다.
- 원격 페이지에 파일 시스템, 셸, 프로세스, 클립보드, 윈도우 생성 권한을 노출하지 않는다.

## 설정과 이전

- 새 설정은 Tauri app config 디렉터리의 `config.json`에 저장한다.
- 쓰기는 같은 디렉터리의 임시 파일에 기록한 뒤 rename하는 방식으로 원자 처리한다.
- 새 설정이 없을 때만 OS별 Electron userData 후보 경로에서 기존 설정을 찾는다.
- 읽은 기존 설정을 현재 스키마로 정규화해 새 위치에 저장하고 원본은 삭제하지 않는다.
- 설정 스키마는 `version: 1`, 슬롯 네 개, `enabled`, `url`, `zoom`을 유지한다.
- 설정 파일이 손상된 경우 기본값을 사용하고 관리 화면에 경고 상태를 전달한다.

## 오류와 복구

- 슬롯 상태는 `idle`, `loading`, `ready`, `error`, `disabled`로 유지한다.
- 페이지 로딩 시작·완료·실패를 슬롯 상태로 전달한다.
- 웹뷰 프로세스 또는 Window가 종료되면 해당 슬롯 또는 전체 출력 구성을 한 번 재생성한다.
- 연속 재생성 실패 시 자동 반복을 중단하고 관리 화면에 오류를 표시한다.
- 모니터 배율·해상도 변경, 절전 복귀 후 전체 레이아웃과 확대율을 다시 적용한다.
- 네트워크 오류 때문에 다른 슬롯 실행을 중단하지 않는다.

## 단일 실행과 종료

- Tauri single-instance 플러그인으로 중복 실행을 방지한다.
- 두 번째 실행 요청은 출력 모드를 중지하고 관리 Window를 표시한다.
- 관리 Window를 닫으면 애플리케이션을 종료한다.
- 앱 종료 시 단축키를 해제하고 모든 child Webview와 Window를 명시적으로 닫는다.

## 테스트 전략

### 자동 테스트

- Rust 단위 테스트: 2×2 물리 좌표, DPI 변환, 확대율, 설정 정규화, 원자 저장, 설정 이전
- 프런트엔드 테스트: 슬롯 편집, 드래그 교환, 저장 직렬화, 실패 상태 표시, 미리보기 제거 확인
- Tauri 통합 테스트: command 호출자 제한, 이벤트 payload, 실행·중지 상태 전이
- WebdriverIO 스모크: 관리 화면 렌더링, 설정 저장, RUN 호출, ESC 또는 오버레이 복귀

### 플랫폼 검증

- macOS 14 이상: Retina 및 비-Retina, simple fullscreen, ESC, 세션 유지
- Windows 10: 100%·125%·150%·200% 배율, WebView2, fullscreen, NSIS 설치본
- Ubuntu 24.04 Wayland/X11: 4분할 위치, GTK 재배치, ESC, 절전 복귀, `.deb` 설치본
- 3840×2160 장비: 각 슬롯 CSS viewport와 실제 영역 1920×1080 확인
- 반복 시험: RUN/복귀 100회 동안 검은 화면·웹뷰 누수 없음
- 장시간 시험: 8시간 실행 후 웹뷰 정지와 지속적인 메모리 증가 없음

## 패키징

- macOS: arm64·x64 앱과 DMG, 서명·공증 설정 가능 구조
- Windows 10: x64 NSIS 설치 파일, WebView2 Evergreen 확인
- Ubuntu 24.04: x64 `.deb`
- CI는 세 운영체제의 네이티브 runner에서 각각 빌드·테스트한다.

## 비범위

- 웹 미리보기 및 스크린샷 캡처
- 출력 모니터 선택 UI
- 자동 업데이트
- Linux 배포판 일반화
- 모바일 지원
- 원격 페이지의 카메라·마이크·위치 권한 지원

## 완료 기준

- 세 운영체제에서 네 원격 웹뷰가 주 모니터를 빈 영역 없이 2×2로 채운다.
- 3840×2160 출력에서 각 슬롯이 1920×1080 영역을 사용한다.
- 설정 확대율 50%부터 150%까지 모든 지원 배율에서 동일하게 동작한다.
- ESC 또는 안내 오버레이로 관리 화면에 항상 복귀할 수 있다.
- 한 슬롯의 로딩 실패가 다른 슬롯 출력을 중단하지 않는다.
- 원격 웹뷰에서 Tauri command와 시스템 권한에 접근할 수 없다.
- 기존 Electron 설정이 최초 실행 시 새 설정 위치로 복사된다.
- 자동 테스트와 세 운영체제 스모크 테스트가 통과한다.
