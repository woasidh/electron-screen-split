# Login Extension Production Logging Design

## 목표

- 로그인 연장 실행 주기를 테스트용 1분에서 운영용 1시간으로 복구함.
- 복잡한 사후 검증 없이 selector 발견 여부와 `click()` 호출 결과만 기록함.
- 실제 staging에서 확인한 기존 selector 조건을 유지함.

## 실행 주기

- RUN 1시간 후 첫 동작을 실행함.
- 이후 출력이 종료될 때까지 1시간마다 반복함.
- 출력 종료, 창 파괴, 앱 종료 시 기존 취소 처리를 유지함.

## Selector

- 표시되고 활성화된 `<button>`만 검사함.
- `stamp`와 `stamp--normal` class를 모두 가져야 함.
- 전달받은 em dash 표기 `stamp—normal`도 허용함.
- 정리한 텍스트가 `^\d{2}:\d{2}:\d{2}$`와 일치해야 함.
- 후보가 정확히 하나일 때만 클릭함.

## 로그와 화면 알림

- 후보 1개 발견 시 `로그인 연장 버튼 발견`을 console info로 기록함.
- `click()`이 예외 없이 반환되면 `로그인 연장 버튼 클릭 완료`를 console info로 기록함.
- click 예외가 발생하면 `로그인 연장 버튼 클릭 실패`를 console error로 기록함.
- 후보 0개 또는 여러 개이면 기존 console warn과 실패 알림을 유지함.
- 기존 우상단 5초 알림을 유지하고 성공 문구를 `로그인 연장 버튼 클릭 완료`로 명확히 함.

## 제외 범위

- 클릭 전후 시간값 비교를 하지 않음.
- 서버 응답이나 네트워크 요청 성공 여부를 확인하지 않음.
- 재시도, MutationObserver, IPC, 파일 로그, 영구 이력 저장을 추가하지 않음.
- 자동 로그인과 계정정보 저장을 추가하지 않음.

## 검증

- Rust 상수가 3600초인지 테스트함.
- 단일 후보에서 발견 로그, click 호출, 완료 로그가 순서대로 발생하는지 DOM 테스트함.
- click 예외에서 실패 로그와 실패 알림이 발생하는지 테스트함.
- 후보 없음, 복수 후보, 숨김·비활성, class 조건 테스트를 유지함.
- 전체 TypeScript·Rust 테스트, 타입 검사, Clippy, production build를 실행함.
- 최신 Tauri 앱을 재실행하고 원격 feature 브랜치만 갱신함.
