# Visible Drag Preview Design

## 문제

- 슬롯 카드는 2×2 그리드의 절반 크기를 채움.
- 웹 엔진의 기본 드래그 이미지는 카드 전체를 캡처해 Ubuntu WebKitGTK에서 창처럼 크게 보였음.
- 1×1 투명 캔버스로 교체한 뒤에는 크기 문제는 사라졌지만 드래그 미리보기도 완전히 사라졌음.

## 결정

- `dragstart`에서 180×72 캔버스에 화면 번호, 위치, URL을 그려 네이티브 드래그 이미지로 지정함.
- 포인터가 미리보기 중앙에 위치하도록 오프셋을 90×36으로 지정함.
- 캔버스는 드래그 시작 시 문서에 추가하고 드롭 또는 `dragend`에서 제거함.
- 원본 카드의 `is-dragging` 반투명 표시와 대상 카드의 `is-drop-target` 테두리는 유지함.

## 범위

- `src/renderer/view.ts`의 드래그 이미지 렌더링만 변경함.
- 교환, 렌더 보류, 설정 저장, 출력 웹뷰 코드는 변경하지 않음.
- 추가 런타임 의존성을 사용하지 않음.

## 검증

- 실제 `dragstart` 이벤트에 모의 `DataTransfer`를 연결해 `setDragImage()` 인자를 검사함.
- 지정 이미지가 180×72 캔버스이고 화면 정보가 그려지는지 확인함.
- 포인터 오프셋이 90×36인지 확인함.
- `dragend` 후 임시 캔버스가 제거되는지 확인함.
- 전체 프런트엔드 테스트, 타입 검사, 빌드, Rust 테스트를 실행함.
