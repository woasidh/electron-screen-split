# Login Extension Design

## 목표

- 화면별로 로그인 연장 사용 여부를 저장함.
- 옵션이 켜진 활성 웹뷰에서 RUN 1시간 후부터 매시간 로그인 연장 버튼을 클릭함.
- 자동 로그인이나 계정정보 저장은 수행하지 않음.

## 설정과 UI

- `SlotConfig`에 `loginExtension: boolean`을 추가함.
- 기존 설정 파일에 필드가 없으면 `false`로 마이그레이션함.
- 기본값은 `false`임.
- 선택 화면 설정 패널에 `로그인 연장` 스위치를 추가함.
- 카드 요약에는 옵션이 켜진 경우 `로그인 연장`을 표시함.
- 카드를 교환하면 URL, 확대율, 사용 상태와 함께 옵션도 이동함.

## 버튼 탐색

- 대상 staging URL은 현재 개발 환경에서 사내 DNS/VPN 미연결로 접근할 수 없었음.
- 옵션이 켜진 웹뷰에서 활성화되고 화면에 표시된 `<button>`만 검사함.
- 버튼은 `stamp`와 `stamp--normal` class를 모두 가져야 하며, 전달받은 em dash 표기 `stamp—normal`도 허용함.
- 공백을 정리한 버튼 텍스트가 `^\d{2}:\d{2}:\d{2}$`와 일치해야 함.
- 일치하는 버튼이 정확히 1개인 경우에만 `click()`을 호출함.
- 0개이거나 여러 개인 경우 아무 동작도 하지 않고 다음 주기에 재시도함.

## 실행 구조

- `WallController`가 로그인 연장 전용 취소 플래그와 스케줄러 스레드를 관리함.
- RUN 완료 후 옵션이 켜진 활성 슬롯의 웹뷰 핸들을 복제해 스케줄러를 시작함.
- 스케줄러는 1시간 대기 후 각 대상 웹뷰에 버튼 클릭 스크립트를 `eval()`함.
- 이후 취소될 때까지 1시간마다 반복함.
- 출력 종료, 창 파괴, 앱 종료 시 스케줄러를 취소함.
- 다시 RUN하면 이전 스케줄러를 취소하고 현재 설정으로 새로 시작함.

## 실패 처리

- 버튼을 찾지 못하거나 여러 개 찾으면 웹 페이지를 변경하지 않음.
- 개별 웹뷰의 스크립트 실행 실패는 다른 화면의 연장 동작을 막지 않음.
- 로그인 만료 후 재로그인, 네트워크 복구, 페이지 새로고침은 범위에 포함하지 않음.

## 검증

- 기존 설정 JSON에 필드가 없어도 `false`로 로드되는지 테스트함.
- 정규화와 카드 교환 시 필드가 보존되는지 테스트함.
- 관리 화면 스위치 렌더링과 저장 경로를 테스트함.
- 스크립트가 단일 시간 버튼만 클릭하도록 브라우저 DOM 테스트를 추가함.
- 짧은 테스트 주기로 스케줄러 실행과 취소를 검증함.
- 전체 TypeScript·Rust 테스트, 타입 검사, Clippy, 빌드를 실행함.
